## 🔴 P1 — importantes

### 1. FAB création rapide masque du contenu (mobile, toutes pages)
- **Régression** : le FAB global violet (`Layout.tsx:297`, `fixed bottom-20 right-4 z-40`) recouvre : le bouton **« Sauvegarder » de Paramètres mobile** (capture `mobile-light/settings-profile-tab.png` — le CTA est à moitié caché), et la **colonne streak du tableau Habitudes** (« 5 🔥❄ » illisible).
- **Pourquoi** : un CTA primaire inaccessible/illisible est une vraie perte fonctionnelle ; en juin le FAB n'existait que sur certaines pages et rien n'était masqué.
- **Correction exacte** : réserver le même padding bas que `HabitsPage.tsx:104` (`pb-[calc(64px+env(safe-area-inset-bottom)+88px)]`) sur **toutes** les pages mobiles où le FAB est monté — `SettingsPage.tsx:306` n'a que `+24px`. Alternative : ne pas monter le FAB sur /settings et /statistics (pages sans création de tâche).
- **Gravité** : P1 · **Temps** : 30–45 min (audit du padding de chaque page + test 4 combos).

### 2. Page Paramètres hors design system (police runtime Google Fonts)
- **Régression** : `SettingsPage.tsx:375`, `settings/DataTab.tsx:52`, `settings/primitives.tsx:19` et `organization/OrganizationSettingsCard.tsx:27` utilisent **Bricolage Grotesque + DM Sans**, chargées **au runtime depuis fonts.googleapis.com**, avec un titre `text-3xl font-extrabold` maison — alors que `src/components/ui/typography.tsx` documente explicitement que Settings doit utiliser `<PageHeading variant="standard">` (Inter, `text-2xl sm:text-3xl font-bold`).
- **Pourquoi** : (a) seule page de l'app avec une autre police → rupture d'identité visible, (b) fetch externe = FOUT + dépendance CSP/`connect-src` (vérifier `vercel.json`), (c) contredit la source de vérité typographique.
- **Correction exacte** : remplacer les h1/h3 par `PageHeading`/`SectionHeading`, supprimer l'injection de `<link>` Google Fonts dans `primitives.tsx`.
- **Gravité** : P1 · **Temps** : 45 min.

### 3. La diff en cours modifie un fichier mort — le travail est invisible
- **Régression** : les modifications non commitées de `src/components/okr-modal/OKRModalDesktopBody.tsx` (unification du bouton catégorie) portent sur un composant **qui n'est plus rendu** : `OKRPage.tsx` utilise `OKRModalSheet` (ligne 16), `OKRModal.tsx` n'est importé nulle part.
- **Pourquoi** : ce n'est pas visuel mais c'est la cause racine des régressions récurrentes — tant que la chaîne morte existe, chaque session IA risque d'éditer le mauvais fichier (c'est arrivé dans la diff actuelle).
- **Correction exacte** : supprimer la chaîne morte (voir §Fichiers morts) et annuler les hunks de `OKRModalDesktopBody.tsx` de la diff en cours ; `OKRModalSheet.tsx:160` a déjà le pattern.
- **Gravité** : P1 (dette qui génère des régressions) · **Temps** : 20 min.

## 🟠 P2 — mineures

### 4. Tableau Habitudes mobile : colonne de jour coupée
- **Régression** : en juin la vue Tableau mobile affichait 4 colonnes de jours complètes ; aujourd'hui une **colonne tronquée en sliver** apparaît à gauche (en-tête « …m. » et cercles coupés en deux) — captures `mobile-*/habits-table-view.png` vs baseline (diff 27 %).
- **Pourquoi** : une colonne coupée en deux fait « rendu cassé » ; c'est l'écran le plus dégradé de tout le diff.
- **Correction exacte** : aligner le scroll horizontal initial sur une frontière de colonne (`scroll-snap-type: x mandatory` + `scroll-snap-align` sur les colonnes, ou calcul de la largeur pour un nombre entier de colonnes). L'icône ❄ (jours joker, commit d4cc7dd) élargit la colonne streak — prévoir sa largeur dans le calcul.
- **Gravité** : P2 · **Temps** : 1–2 h.

### 5. Trois patterns différents pour « créer une catégorie depuis un modal »
- **Régression** : même action, trois rendus : **TaskModal** (`DesktopDetailsStep.tsx`) bouton texte `+ Ajouter` (Plus 12, label `mb-2`) ; **OKRModalSheet** (l.161) même bouton mais contexte `Label` shadcn ; **EventModal** (`EventModalFormDesktop.tsx:402`) une **icône `Plus w-6 h-6` nue posée dans le `<label>`** avec `hover:scale-125`, sans élément bouton → pas de focus clavier, pas d'aria-label, hover différent.
- **Pourquoi** : deux composants jumeaux au rendu différent = exactement le genre de détail qui casse l'impression premium ; la variante EventModal est en plus une régression a11y (cible non focusable).
- **Correction exacte** : extraire un composant `AddCategoryButton` (bouton texte `+ Ajouter`, `text-xs font-semibold text-blue-600 dark:text-blue-400`, Plus 12) et l'utiliser aux trois endroits.
- **Gravité** : P2 · **Temps** : 45 min.

### 6. Hover du bouton « Ajouter » illisible en dark
- **Régression** : `hover:text-blue-700` sans variante dark (`DesktopDetailsStep.tsx`, `OKRModalSheet.tsx:164`, + copie morte) : en dark le texte passe de `blue-400` à `blue-700` au survol → **le contraste baisse au hover** (inverse du feedback attendu).
- **Correction exacte** : `hover:text-blue-700 dark:hover:text-blue-300`.
- **Gravité** : P2 · **Temps** : 10 min.

### 7. Sidebar Agenda : affordances disparues des cartes tâches
- **Régression** : en juin chaque carte « Tâches disponibles » affichait le lien **« ↗ Glisser vers le calendrier »** et un menu **« ⋯ »** ; les deux ont disparu (baseline `desktop-light/agenda-default.png` vs actuel).
- **Pourquoi** : le drag vers le calendrier n'est plus découvrable ; les actions du menu ⋯ ne sont plus accessibles depuis la sidebar. Aucun commit « feat » ne revendique cette suppression → régression probable lors d'un refactor.
- **Correction exacte** : restaurer le hint et le menu ⋯ (ou confirmer la suppression volontaire et l'acter dans docs/UI-PATTERNS.md).
- **Gravité** : P2 · **Temps** : 30–60 min.

### 8. Événement « Sport » rendu bord-à-bord sans rayon (mobile, vue Jour)
- **Régression** : dans `mobile-*/agenda-default.png`, « Formation React » a des coins arrondis et une marge droite ; « Sport » s'étale jusqu'au bord droit, coins carrés. Deux événements du même calendrier, deux rendus.
- **Pourquoi** : incohérence de cartes visible au premier écran de l'agenda mobile.
- **Correction exacte** : même inset/rayon pour tous les events (probablement un event récurrent ou multi-slots qui prend un chemin CSS FullCalendar différent — inspecter les classes `fc-event-*` de cet event).
- **Gravité** : P2 · **Temps** : 30–60 min.

### 9. Placeholder du quick-add tronqué (mobile)
- **Régression** : le champ NLP affiche « Ex : Préparer la réunio » coupé (`mobile-*/tasks-add-task-form.png`).
- **Correction exacte** : placeholder court sur mobile (« Ex : Réunion demain 10h ») ou largeur de champ accrue.
- **Gravité** : P2 · **Temps** : 10 min.

### 10. Écrans « stats » mobiles : libellé tronqué
- **Régression** : « 365 derniers jo... » tronqué dans la carte temps investi (`mobile-*/statistics-default.png`).
- **Correction exacte** : « 365 jours » / « 1 an », ou largeur de colonne libellé suffisante pour le plus long libellé.
- **Gravité** : P2 · **Temps** : 10 min.
